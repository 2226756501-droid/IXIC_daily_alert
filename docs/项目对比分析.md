# IXIC（纳斯达克监控） vs Bishe（气象预测）— 对比分析

## 一、总体对比

| 维度 | IXIC | Bishe |
|------|------|-------|
| 项目类型 | 数据监控系统（实时数据采集+告警） | 预测分析系统（机器学习建模） |
| 核心算法 | Z-score（纯数学，无第三方 ML 库） | Facebook Prophet（时间序列预测） |
| 交互方式 | Streamlit 网页 + 邮件推送 | Streamlit 网页 + CLI 命令行 |
| 自动化程度 | GitHub Actions 全自动，无人值守 | 纯本地，手动运行 |
| 代码风格 | 函数式 + 模块化 | 面向对象（类） + 配置驱动 |
| 使用难度 | 部署后不用管，每天自动跑 | 每次需要手动执行 main.py |

## 二、代码质量对比

| 指标 | IXIC | Bishe |
|------|------|-------|
| **测试** | 47 个 pytest 测试 ✅ | 0 个测试 ❌ |

| **依赖管理** | requirements.txt 版本宽松 ✅ | pandas<2.0 过时限制 ❌ |
| **Docker** | 多阶段构建 ✅ | 无 Dockerfile ❌ |

## 五、Bishe 可以借鉴 IXIC 的地方

1. **加测试** — 至少给数据处理和模型训练写 pytest
2. **加日志** — 把 print 换成 logging，方便排错
3. **加 CI** — GitHub Actions 自动跑测试，防止改坏
4. **加缓存保护** — 数据加载失败时用本地缓存兜底
5. **统一环境变量** — 不要在 4 个模块里重复写 `os.environ['TMP']`
6. **去掉 pandas 版本上限** — `pandas<2.0` 已经过时了
