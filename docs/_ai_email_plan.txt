让 AI 写邮件 - 功能规划文档
一、当前邮件长什么样?
现在的邮件内容靠 Python 字符串拼接, 结果是干巴巴的数字描述, 没有任何分析和可读性。
标题示例: 纳斯达克数据 2026-06-14 涨跌幅 -1.23%
正文示例: 纳指 19876.54, 涨跌 -247.32 (-1.23%), Z-score -2.38
异常内容是硬编码拼接, 缺乏人味。
二、改完的邮件长什么样?
改完后, 邮件由 AI 自动生成, 风格如下:
标题: 纳斯达克日报 6月14日 收涨 +0.31%, 市场平稳
正文:
  今日纳斯达克收报 25,888.84 点, 较前日上涨 79.14 点 (+0.31%)。
  关键指标:
    Z-score: 0.05 -> 正常波动
    连跌天数: 0
    市场状态: 正常
  市场分析:
    今日小幅收涨, Z-score 接近零, 属于典型窄幅震荡。
  相关新闻:
    - 高收益储蓄利率最高达 4.1% APY
    - 抵押贷款利率整体走低
  综合判断:
    短期内无系统性风险, 建议继续持有观察。
异常时会有额外段落:
  异常提醒: 纳指连跌 3 天, Z-score -2.38, 已标记异常时段。
  历史参考: 上次类似情况出现于 5月12日, 后续 2 天内反弹 +1.2%。
三、具体需要改哪里?
3.1 nasdaq.py 需要改的部分
改动量极小, 只替换邮件标题和正文的拼接逻辑。
a) 在文件开头添加导入:
from modules.agent_engine import chat
b) 删除原有的 subject/body 拼接代码, 替换为 AI 生成:
    构建一个 prompt, 把当天所有数据传给 AI
    AI 生成邮件全文
    第一行作为标题, 其余作为正文
c) 原有的异常判断逻辑 (state 切换, memory 记录等) 全部保留不动。
    只有邮件生成那一部分换掉。
3.2 agent_engine.py 需要改的部分
不需要改。现有的 chat() 函数已经可以直接调用。
不过可以新增一个专门的 generate_email() 函数, 让调用更清晰。
3.3 测试需要改的部分
新增 test_agent_email.py
测试 generate_email() 返回格式是否正确
测试没有 API key 时是否优雅降级
四、风险与应对
风险: AI 调用失败 (网络/API 超时)  -> 应对: catch 异常, 降级为现有的模板拼接, 保证邮件总能发出去
风险: AI 生成的标题太长  -> 应对: 限制标题长度, 超过 50 字就截断
风险: AI 跑题  -> 应对: 在 prompt 里加严格的格式要求
风险: API 费用  -> 应对: 每次邮件约 500 tokens, 约 0.01 分钱, 一年不到 5 分钱
风险: 余额不足  -> 应对: 检测 402 错误, 自动降级为模板邮件
五、实施步骤
1. 在 agent_engine.py 里新增 generate_email() 函数
2. 修改 nasdaq.py: 导入 generate_email, 替换邮件拼接逻辑
3. 手动运行 nasdaq.py, 验证邮件内容
4. 确认正常后, 提交代码 push
5. 观察下一次 GitHub Actions 自动运行时邮件是否正常
六、如果改坏了怎么恢复?
方法 1: 在 nasdaq.py 里注释掉 AI 生成的代码, 取消注释原有的模板代码
方法 2: git revert 回退到改动前的版本
两种方法都不影响其他功能 (数据采集、异常检测、网页展示等)。
