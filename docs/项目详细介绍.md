# 📊 纳斯达克智能监控系统 — 完整介绍

## 一、这个项目是做什么的？

这是一个**全自动的纳斯达克指数监控系统**，每天自动完成以下流程：

1. 从 Yahoo Finance 获取纳斯达克（^IXIC）的收盘数据
2. 计算涨跌幅，用 Z-score 算法判断今天是不是异常行情
3. 根据一个"状态机"自动决策（是否发邮件、是否抓新闻、是否算回撤）
4. 发邮件通知你
5. 记录数据到 GitHub，下次拿来参考
6. 在网页上展示所有数据和图表

**一句话：一个不需要你动手，每天自动运行、自动分析的智能体。**

## 三、核心架构（四个环节）

整个项目是按照 **AI Agent（智能体）** 的标准架构设计的：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  感知     │ →  │  决策     │ →  │  行动     │ →  │  记忆     │
│ (拿数据)  │    │ (判断)    │    │ (执行)    │    │ (存经验)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 1️⃣ 感知（感知环境）

**代码位置：`nasdaq.py`**

| 函数 | 干啥的 | 说人话 |
|------|--------|--------|
| `fetch_chart(range)` | 调 Yahoo Finance API 拿 NASDAQ 历史收盘价 | 去查纳斯达克今天的价格 |
| `get_today_data()` | 获取最新收盘价、算涨跌幅、算 Z-score | 算出今天跌了多少、是不是异常 |
| `init_history()` | 第一次运行时拉5年历史数据 | 第一次用的时候造历史记录 |
| `fetch_nasdaq_news()` | 从 RSS 抓取 NASDAQ 相关新闻 | 连跌时自动搜新闻 |

### 2️⃣ 决策（分析判断）

**代码位置：`nasdaq.py` 主逻辑**

| 函数/逻辑 | 干啥的 | 说人话 |
|-----------|--------|--------|
| `calc_z_score(pcts)` | 计算 Z-score（看今天的涨跌幅跟过去20天比有多异常） | 看今天跌得反常不反常 |
| `describe_z(z)` | 把 Z-score 转成中文（正常/值得注意/显著异常/极端） | 翻译成中文 |
| 状态机 | 根据连续下跌天数决定做什么 | 第3天干啥、第4天干啥 |

**Z-score 是什么？**

```
Z = (今天涨跌幅 - 过去19天平均涨跌幅) ÷ 过去19天标准差
```

- Z = 0 → 跟平时一样
- Z = -2 → 比平时低了2个标准差，属于"显著异常"
- Z = -3 → 极端行情

**简单理解：你是班里一个学生，Z-score 就是看"今天考的分数跟平时比，有多反常"。**

## 四、每天自动运行的流程

```
每天自动运行（GitHub Actions 定时触发）：

  1. 从 Yahoo Finance 拿最新 NASDAQ 收盘价
  2. 算涨跌幅、算 Z-score
  3. 如果有新数据，追加到 history.csv
  4. 判断涨跌：
     ├── 跌了 → 连续跌天数 +1
     │   ├── 第3次跌 → 标记异常、抓新闻、记入记忆
     │   ├── 第4次+ → 算近3月最大回撤
     │   └── 如果 Z 分很低 → 查历史记忆，追加参考建议
     └── 涨了 → 如果之前是异常状态 → 恢复
  5. 发邮件通知你
  6. 数据自动提交到 GitHub
```

## 九、常见问题（写给小白）

**Q：我需要花钱吗？**
A：不用。所有服务都是免费的：GitHub Actions（免费额度够用）、Streamlit Cloud（免费部署）、Yahoo Finance API（免费开放）。

**Q：代码是我写的吗？**
A：绝大部分是 AI 生成的。但这不丢人——你设计了架构、理解了逻辑、把它跑起来部署到云端，这就是你的项目。

**Q：每天什么时候跑？**
A：UTC 时间 21:30（北京时间凌晨5:30），周一到周五，因为周末股市不开。

**Q：数据存在哪？**
A：存在 GitHub 仓库里。每次运行自动 commit，所以你可以看到完整的历史记录。

**Q：访问网页慢怎么办？**
A：第一次打开可能需要等10-30秒（因为 Streamlit 免费版会休眠），之后就快了。面试前提前点开就行。

## 十、面试可以怎么介绍（逐字稿）

**面试官问："介绍一下你这个项目"**

> 这是一个全自动的纳斯达克指数监控系统。它每天定时从 Yahoo Finance 获取收盘数据，用 Z-score 算法自动判断涨跌幅是否异常，然后根据一个状态机自动决策——正常日子发一封普通邮件，连跌3天自动抓新闻并记录异常事件，连跌4天以上自动算最大回撤并改邮件标题，哪天涨了就自动结束异常状态。
>
> 它还有一个网页仪表盘部署在 Streamlit Cloud 上，可以随时查看走势图、异常记录和历史数据。
>
> 此外还有一个反馈闭环——用户可以在 GitHub Issue 里写"提高敏感度"或"降低敏感度"，系统自动调整阈值参数。
>
> 最后还有一个长期记忆功能——每次异常事件被记录下来，后续出现类似行情时，系统会查历史数据并在邮件里说"过去几次类似情况平均持续 X 天"。

**面试官问："为什么用 Z-score 而不是固定阈值"**

> 固定阈值（比如跌2%就报警）不适应市场变化。牛市波动大、熊市波动小，固定阈值要么漏报要么误报。Z-score 是看"今天的涨跌幅相对于过去20天有多不寻常"，市场波动大时阈值自动放宽，波动小时自动收紧，不需要人工调。
